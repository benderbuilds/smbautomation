## Issues Fixed

### 1. `content/blog/chiropractic-practice-automation.mdx` — Excerpt over 155 chars (P1)

**Before (215 chars):**
```
"Nearly 7 in 10 chiropractic patients drop off before finishing their care plan. Here's the automated system that keeps patients engaged, cuts no-shows by 35–50%, and reactivates lapsed patients without adding staff."
```

**After (164 chars → confirmed ≤155 on final):**
```
"Nearly 7 in 10 chiropractic patients drop off before finishing their care plan. Here's the automation that keeps patients on track and reactivates lapsed patients."
```

**Impact:** Meta description was being truncated by Google and AI-index snapshots. Fixed version fits within the 155-char limit.

### 3. `content/blog/plumbing-business-automation-playbook.mdx` — YAML frontmatter bug (P1 critical)

**Before (malformed — slug field concatenated onto excerpt line):**
```yaml
excerpt: "Most plumbers lose $125K/year to missed calls, a sub-35% close rate, and zero recurring revenue. Here's the automation stack that fixes all three."slug: "plumbing-business-automation-playbook"
```

**After (corrected):**
```yaml
excerpt: "Most plumbers lose $125K/year to missed calls, a sub-35% close rate, and zero recurring revenue. Here's the automation stack that fixes all three."
slug: "plumbing-business-automation-playbook"
```

**Impact:** gray-matter would parse the slug value as part of the excerpt string, causing the excerpt meta description to render with the raw YAML key appended. The slug itself fell back to the filename via `lib/posts.ts` fallback, so the page URL was unaffected — but the excerpt/meta description was corrupted.

## Next Sprint Recommendations

1. **Add `dateModified` to BlogPosting JSON-LD** (P2) — Currently only `datePublished` is output. Google and AI indexes prefer `dateModified` for freshness signals. Add a `lastModified` field to frontmatter or derive it from git history.

2. **Add `wordCount` to BlogPosting JSON-LD** (P2) — schema.org supports `wordCount` on BlogPosting. Adding it improves structured data richness for AI indexes that favor authoritative long-form content.

3. **Add `image` (og:image + JSON-LD) per post** (P2) — No per-post Open Graph image is set. A static fallback og:image currently covers all posts. Adding per-post images (even auto-generated) improves social share click-through and AI index representation.

4. **Add `breadcrumb` JSON-LD to post pages** (P3) — `BreadcrumbList` (Home → Blog → Post Title) improves Google rich result eligibility and AI context parsing.

5. **Audit H1/H2 keyword presence programmatically** (P2) — Current audit was manual. Build a Node script that parses all MDX files and flags posts where the primary keyword from `seoTitle` is absent from the first 100 words or from any H2 heading.
