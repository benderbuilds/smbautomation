## Issues Found by Category

### ON-PAGE SEO

#### ✅ FIXED — Excerpts over 155 characters (3 posts)
Meta description = excerpt field. Google truncates at ~155 chars; over-length excerpts display poorly in SERPs and AI summaries.

| File | Old Length | New Length |
|------|-----------|-----------|
| `auto-repair-shop-automation-playbook.mdx` | 165 chars | 142 chars |
| `maintenance-agreement-automation.mdx` | 174 chars | 141 chars |
| `roofing-contractor-automation-playbook.mdx` | 155 chars | 151 chars |

#### ✅ FIXED — Frontmatter bugs: slug field on same line as excerpt (2 posts)
`gray-matter` was receiving a malformed YAML value for `excerpt` (including the slug key-value appended to the string). The `slug` field was missing its own line, so it defaulted to the filename (functionally OK) but the excerpt displayed incorrectly in meta tags and blog index cards.

- `electrician-business-automation-playbook.mdx` — fixed
- `pest-control-business-automation-playbook.mdx` — fixed

#### ✅ FIXED — Missing FAQ sections (3 posts)
Posts missing `## Frequently Asked Questions` before the CTA. FAQ sections are a primary GEO signal — AI assistants extract Q&A pairs directly for featured answers.

- `auto-repair-shop-automation-playbook.mdx` — added 3 Q&As
- `new-customer-onboarding-automation-service-business.mdx` — added 3 Q&As
- `pest-control-business-automation-playbook.mdx` — added 3 Q&As

### SCHEMA / STRUCTURED DATA (GEO)

#### ✅ FIXED — Organization JSON-LD missing from homepage
`app/(site)/page.tsx` had no JSON-LD. Added `schema.org/Organization` with `name`, `url`, and `description`. This is the primary entity-association signal for AI assistants.

```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "SMB Automation",
  "url": "https://smbautomation.io",
  "description": "SMB Automation builds custom AI automation systems for small and mid-size businesses..."
}
```

#### PASS — BlogPosting JSON-LD on post renderer
`app/(site)/blog/[slug]/page.tsx` already outputs correct `schema.org/BlogPosting` with `headline`, `description`, `datePublished`, `author`, `publisher`, and `url`. ✓

## Issues Not Fixed — Manual Attention Required

### P1 — Homepage lacks a blog index page link in sitemap metadata priority
`app/sitemap.ts` sets the blog index at `priority: 0.8` while `/services` is `0.9`. Given that blog posts are the primary GEO content surface, the blog index should be elevated to `0.9` or equal to `/services`. Low effort, high signal to crawlers.

### P2 — Several posts have only 1 incoming link (not 0, but low authority)
The following posts now have exactly 1 inbound link — acceptable per the minimum threshold but worth strengthening in the next sprint:
- `cleaning-business-automation-playbook` (1 link from: `cost-of-manual-processes`)
- `hrt-symptom-tracking-automation` (1 link)
- `landscaping-business-automation-playbook` (1 link)
- `plumbing-business-automation-playbook` (1 link)
- `real-estate-agent-lead-nurture-automation` (1 link)
- `semaglutide-onboarding-automation` (1 link)
- `trt-lab-compliance-automation` (1 link)
- `trt-refill-automation` (1 link)

**Recommendation:** Create a "Healthcare Automation" hub post or a "Playbook Index" post that links to all vertical-specific playbooks in one place.

### P3 — No image/OG image set for blog posts
`app/(site)/blog/[slug]/page.tsx` does not include an `og:image` in the OpenGraph metadata. AI crawlers and social sharing both benefit from a defined image. Consider adding a fallback OG image (e.g., the SMB Automation logo on a dark background) referenced in the site metadata defaults.

## Recommendations for Next Sprint

1. **Create a "Playbook Index" hub post** — a single page linking to all 20+ industry-specific automation playbooks. This would simultaneously resolve the low-authority posts problem and create a high-authority anchor page for the playbook cluster.

2. **Add OG image to blog posts** — Even a single branded fallback image (`/og-default.png`) would improve click-through on social shares and improve AI assistant citation confidence.

3. **Elevate blog sitemap priority** — Change `priority: 0.8` → `priority: 0.9` for the blog index in `app/sitemap.ts`.

4. **Consider a Healthcare hub post** — The 6 healthcare posts (TRT, HRT, GLP-1, semaglutide, med spa, dental) are a strong cluster but not cross-linked well enough. A hub post or a "Healthcare Practice Automation" overview page would strengthen the cluster's topical authority with AI search engines.

5. **Date freshness** — 8 posts are dated March 2026. Consider refreshing stats or adding a "Updated" date field to signal recency to crawlers. AI assistants weight recent content more heavily for factual citations.
