### P2 (Important — Address This Sprint)

**P2-1: Open Graph image missing on all blog posts**  
`app/(site)/blog/[slug]/page.tsx` generates OG metadata without `og:image`. AI search engines and social platforms require an image for rich link previews. Without it, posts shared on LinkedIn, Slack, or surfaced in Perplexity won't show an image preview.  
**Fix:** Add `openGraph.images` to `generateMetadata()` — either a static branded fallback or a dynamically generated image per post using Next.js `ImageResponse`.

**P2-2: Blog index page (`/blog`) missing Open Graph image**  
Same issue as above for the index page.  
**Fix:** Add `openGraph.images` to the static `metadata` export in `app/(site)/blog/page.tsx`.

**P2-3: No `dateModified` field in JSON-LD BlogPosting schema**  
The `BlogPosting` schema in `app/(site)/blog/[slug]/page.tsx` includes `datePublished` but not `dateModified`. Google and AI crawlers use `dateModified` to determine content freshness.  
**Fix:** Add a `dateModified` field to the `PostMeta` interface in `lib/posts.ts`, add an optional `updatedDate` frontmatter field to posts, and surface it in the JSON-LD output (falling back to `datePublished` if not set).

### P3 (Nice to Have — Next Sprint)

**P3-1: No `og:image` in post frontmatter**  
Currently there's no per-post image defined anywhere. For high-traffic posts (speed-to-lead, ai-receptionist, hvac-automation-seasonal-surge), a custom OG image would meaningfully improve CTR from social and AI-cited links.  
**Recommendation:** Design a branded template (e.g., Vercel OG image generation or a static set of 4–5 tag-based banners) and wire it to the `tag` field already present in all post frontmatter.

**P3-2: Blog index title tag is generic**  
`app/(site)/blog/page.tsx` sets `title: 'Blog'` — just the word "Blog". This wastes the title tag for the second most important page on the site.  
**Recommendation:** Change to: `"Automation Insights for Small Business | SMB Automation Blog"` (under 60 chars).
