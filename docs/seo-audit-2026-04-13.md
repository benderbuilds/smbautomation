## Issues Not Fixed (Requiring Manual Attention)

### P1 — No issues at P1 severity remain.

### P2 — Low priority, manual review recommended

**og:image missing on blog posts and homepage.**  
`app/(site)/blog/[slug]/page.tsx` and `app/(site)/page.tsx` do not set `openGraph.images`. AI crawlers and social sharing previews use a generic image or none. Recommended: add a default OG image (1200×630 PNG) and optionally a per-post image in frontmatter.

**Blog index page metadata is generic.**  
`app/(site)/blog/page.tsx` uses title: `'Blog'` which renders as just "Blog" in the tab and in search. Recommended: set `title: 'Automation Blog | SMB Automation'` or similar.

**No `dateModified` on BlogPosting schema.**  
The JSON-LD schema uses `datePublished` only. Adding `dateModified` (even if equal to `datePublished` until a revision is made) improves freshness signals for AI indexers and Google Rich Results.

**No `image` property on BlogPosting schema.**  
Google recommends including an `image` in BlogPosting schema. Without it, articles are ineligible for certain rich result formats.

### P3 — Future sprint consideration

**No breadcrumb schema.**  
Adding `BreadcrumbList` JSON-LD to blog post pages (`Home > Blog > [Post Title]`) improves navigation signals for search and AI systems.

**No `WebSite` schema with `SearchAction`.**  
Adding a `WebSite` schema on the homepage with a `SearchAction` property enables Google's sitelinks search box and provides AI assistants with structured information about the site's search capability.

**Blog index has no canonical tag in code.**  
`app/(site)/blog/page.tsx` does set `alternates.canonical` correctly, but it is worth auditing whether the Next.js layout also emits a duplicate or conflicting canonical in the `<head>`.

**No `hreflang` tags.**  
Not currently an issue (English-only site), but worth noting for future international expansion.

## Recommendations for Next Sprint

1. **Add OG images.** Create a default 1200×630 OG image for the site and add an `openGraph.images` entry in both the blog post and homepage metadata. This directly affects click-through rate from social shares and AI-generated cards.

2. **Add `dateModified` to BlogPosting schema.** Update `lib/posts.ts` to support an optional `updatedDate` frontmatter field; fall back to `datePublished` if not set.

3. **Improve blog index metadata.** Change `title: 'Blog'` to a keyword-targeted, descriptive title. Suggested: `'SMB Automation Blog — Practical AI & Automation Guides for Small Business'`.

4. **Add breadcrumb JSON-LD.** Wire up a `BreadcrumbList` in `app/(site)/blog/[slug]/page.tsx` matching the page hierarchy.

5. **Audit the GLP-1 / HRT content cluster for E-E-A-T signals.** The healthcare posts are high-value GEO targets but would benefit from explicit author credentials or a disclosure that content is reviewed by qualified practitioners. AI assistants (Perplexity, Gemini) increasingly filter medical content against E-E-A-T signals.

6. **Create a content hub page for each vertical.** `/blog/healthcare`, `/blog/hvac`, `/blog/law-firms` — thin category pages that link to all posts in that vertical and include a one-paragraph topical overview. These pages become high-value targets for AI citation of "best resources on X."

7. **Add `publisher.logo` to BlogPosting schema.** Schema.org's AMP/rich-results validator recommends including a `logo` object inside the `publisher`. Add an `ImageObject` with the site logo URL.
