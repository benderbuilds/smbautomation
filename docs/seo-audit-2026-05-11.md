## Issues NOT Fixed (Manual Attention Required)

### P1 — Schema Enhancement
**Add `image` property to BlogPosting JSON-LD.**  
Google's Rich Results Test treats a missing `image` field as a blocker for article rich snippets in Google Discover. None of the 29 posts currently include an OG image or a `image` property in the JSON-LD. This requires either: (a) adding a static fallback image URL to the BlogPosting schema in `app/(site)/blog/[slug]/page.tsx`, or (b) adding per-post `image` frontmatter and threading it through. Estimated effort: 1 hour dev + image asset creation per post or a single fallback.

### P2 — Blog Index Metadata
**Improve the blog index page title tag.**  
`app/(site)/blog/page.tsx` uses `title: 'Blog'` — a generic single-word title with no brand or keyword. Recommended change:  
```ts
title: 'SMB Automation Blog | Automation Insights for Local Businesses'
```
This is a safe, low-effort change that improves click-through from organic results.

### P2 — `dental-practice-automation` H1 Keyword Gap
The H1 is `"What Dental Practices Lose Before a Patient Sits in the Chair"` which does not contain the word "automation." The seoTitle (`"Dental Practice Automation: Fix 4 Revenue Leaks"`) correctly contains the keyword but Google reads the visible H1, not the `<title>` tag, for content relevance. The H2 `"The Tools That Run Dental Practice Automation"` compensates, but an H1 that includes "automation" would strengthen the signal. Recommended: change H1 to `"What Dental Practice Automation Saves Before a Patient Sits in the Chair"` — but this risks changing a strong click-through hook, so flagged for editorial review rather than automated change.

## Recommendations for Next Sprint

1. **Add Open Graph images** — Create a single `og-default.png` (1200×630) branded image and reference it in the BlogPosting JSON-LD `image` field and in `generateMetadata` as `openGraph.images`. This unlocks Google Discover and Twitter/LinkedIn card previews for all posts.

2. **Improve blog index title** — One-line change to `app/(site)/blog/page.tsx` metadata. See P2 above.

3. **Build a topic cluster hub page** — The 29 posts now cover 6 clear topic clusters (Healthcare/Weight Loss, HVAC, Plumbing, Landscaping, Real Estate, Strategy). Creating a `/blog/[topic]` index or a `## Related Posts` section at the bottom of each post strengthens topical authority signals for GEO and improves time-on-site.

4. **Add `dateModified` to BlogPosting schema** — Currently only `datePublished` is in the JSON-LD. Adding `dateModified` (keyed off last git commit touching the file) signals freshness to search and AI crawlers.

5. **Add sitelinks schema / breadcrumb JSON-LD** — `BreadcrumbList` schema on blog posts (`Home > Blog > [Post Title]`) improves snippet display in Google Search and is a recommended GEO signal for content hierarchy.

6. **Review `hrt-patient-reactivation` H2 structure** — The post's H2s are narrative rather than keyword-targeted (`"The Math on Reactivation vs. Acquisition"`, `"Why HRT Patients Cancel — and Why Most Come Back"`). Adding a keyword-bearing H2 like `"The Automated HRT Win-Back System"` earlier in the post would strengthen topical relevance signals without changing body copy.
