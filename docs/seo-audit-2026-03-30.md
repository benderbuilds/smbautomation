## Items Requiring Manual Attention

### P1 — None

### P2 — Medium Priority

| # | Item | Notes |
|---|------|-------|
| M1 | `app/blog/page.tsx` has no `canonical` or `og:url` meta tag | The `metadata` export sets `title` and `description` but no canonical. Add `alternates: { canonical: 'https://smbautomation.io/blog' }` to the metadata object. |
| M2 | No `og:image` set on any post | `generateMetadata` in `app/blog/[slug]/page.tsx` does not include `openGraph.images`. Adding a branded fallback image (1200×630) and including it in the OG tags would significantly improve social share previews and AI crawler card generation. |
| M3 | Blog index page (`/blog`) lacks structured data | Consider adding `ItemList` or `CollectionPage` JSON-LD to `app/blog/page.tsx` to help AI assistants enumerate blog content. |

### P3 — Low Priority / Next Sprint

| # | Item | Notes |
|---|------|-------|
| P3-1 | No `dateModified` in `BlogPosting` schema | The `PostMeta` type has no `updatedAt` field. Add an optional `updatedAt` frontmatter field and use it in the JSON-LD. |
| P3-2 | No `BreadcrumbList` JSON-LD | The breadcrumb nav rendered in `app/blog/[slug]/page.tsx` is visual only. Adding matching JSON-LD would help search engines and AI crawlers understand site hierarchy. |
| P3-3 | `hrt-patient-reactivation-automation` seoTitle could include "automation" | Current: "HRT Win-Back: Reactivate Cancelled Patients" — consider "HRT Patient Win-Back Automation" for keyword alignment. |
| P3-4 | No `sameAs` property in Organization schema | Adding links to LinkedIn, Twitter/X, or other verified profiles in the `Organization` JSON-LD strengthens entity disambiguation for AI assistants. |
| P3-5 | Post keyword density on `bottleneck-audit` is thin | "Bottleneck audit" appears in title and one H2 but only once in body text. Consider reinforcing in the first 50 words. |

## Recommendations for Next Sprint

1. **Add `og:image` to all posts** — a branded 1200×630 fallback image will improve click-through from social previews and AI-generated cards.
2. **Add `BreadcrumbList` JSON-LD** to `app/blog/[slug]/page.tsx` matching the existing visual breadcrumb.
3. **Add `ItemList` schema to `/blog` index page** listing all post titles and URLs.
4. **Add `canonical` to `/blog` page metadata** — currently missing from `app/blog/page.tsx`.
5. **Expand internal linking to 3+ links per post** where topically appropriate — particularly the healthcare cluster (HRT ↔ TRT ↔ GLP-1) which has strong cross-linking opportunity.
6. **Add `updatedAt` frontmatter** to posts when content is refreshed, and wire it into `BlogPosting.dateModified` in the schema.
7. **Consider a pillar page** for each vertical (Healthcare Automation, Local Services Automation, Real Estate Automation) that links to all posts in its cluster — this strengthens topical authority signals for both traditional SEO and GEO.
